# 客户端独有能力

本章会整理 `Projects`、记忆、长文档和系统集成等客户端或账号体系中的进阶能力。

### 客户端能力的共同点

很多进阶能力依赖账号、客户端或会员功能，比如项目空间、记忆、长文档、快捷键、截图、语音和系统集成。不同平台名称不同，思路相近：它们都在增加上下文，或者让 AI 接触更多系统能力。

### `Projects` / 项目空间

**项目空间适合长期任务**。它解决的问题是“背景重复”。如果你正在写论文、学一门课，或者维护一个文档项目，项目空间可以把目标、资料和写作风格固定下来，让后续对话继续沿用。

但它也带来一个新问题：**背景会沉淀**。旧资料、旧目标、旧偏好如果没有清理，AI 可能继续按它们回答。

更好的做法是：

- 当前任务需要的资料，放进去；
- 已经过期的资料，删掉；
- 固定风格和目标，写清楚；
- 临时想法，先留在普通对话里观察。

### 记忆

记忆功能适合保存长期偏好，比如你喜欢中文回答、希望先简后详、常用 `Windows` 和 `PowerShell`。这些偏好每次都重复说会很麻烦，适合交给记忆。

普通对话里的信息通常只影响当前上下文；记忆里的信息可能在你之后问完全不同的问题时仍然被拿来参考。如果它记住了错误偏好、过期身份或敏感背景，后续回答就可能持续偏离。

例如你曾经为了一个临时任务说“把我当成初中生解释”，如果这变成长期记忆，之后你问专业问题时，回答可能一直过度简化。再比如你让它记住某个公司内部项目名，之后在不相关对话里被提到，就可能造成不必要的信息暴露。

所以，**长期稳定的偏好可以放进记忆，敏感信息、临时身份和短期任务要求更适合留在当前对话里**。

### 长文档能力

长文档能力适合处理报告、论文、书稿、课程资料、项目文档。

处理长文档时，顺序应该是**先让 AI 建立目录感，再让它分析局部**。也就是说，先问“这份文档大体分成哪几部分”，再问“第 3 部分的核心论点是什么”，最后再让它提炼结论。材料越长，越适合这样分层处理。

重要结论仍然要回到原文核对。长文档能力适合帮你快读，最终审稿仍然需要你自己完成。

### 系统集成

系统集成包括截图、剪贴板、文件访问、快捷键、语音、日历、邮件、IDE 等。它们能提高效率，也意味着更高权限。

**权限的本质是“它能接触到什么”**。一个只能接收你手动输入文字的网页工具，看到的信息很有限；一个拥有截图、剪贴板、文件夹访问或邮件访问权限的客户端，能接触到的信息就多得多。能力越强，误操作或配置错误带来的影响也越大。

使用原则：

```text
长期任务放项目。
长期偏好放记忆。
敏感材料先脱敏。
重要结论回原文。
系统权限按需开启。
```
