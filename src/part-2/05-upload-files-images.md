# 上传文件、图片、截图来提问

本章会介绍上传文档、图片和截图后的常见问法，以及需要注意的隐私边界。

### 上传材料后，问题会变具体

只用文字提问时，AI 主要靠你描述背景。上传文件、图片或截图后，它可以直接基于材料回答，效果通常更好。

这里的关键是 **上下文**。你只说“这个报错怎么办”，它只能猜；你上传截图后，它能看到报错文字、软件界面、路径、按钮状态等线索，判断就会具体很多。

上传的材料会成为这次对话的一部分。不同平台对数据保存、审核、训练使用、日志保留的政策不同。作为普通用户，最稳妥的原则是：只上传你愿意交给这个平台处理的内容。

最常见的三类材料就够你先练习：一是文档，比如文章、PDF、讲义；二是图片或截图，比如报错页面、说明书、软件界面；三是结构化数据，比如表格或一段日志。材料类型不同，提问方式也应该不同。

### 读文档的问法

上传文档后，直接说明你要用这份文档做什么。只说“看看”太模糊，AI 不知道你要摘要、找问题、改写还是出题。

```text
请阅读这个文件，输出：
1. 300 字以内摘要
2. 5 个关键观点
3. 3 个我需要进一步核查的问题
不要添加文件里没有的信息。
```

如果是学习资料：

```text
请把这份材料整理成适合复习的笔记。
包括：核心概念、容易混淆的点、5 道自测题。
```

### 看截图的问法

截图适合处理难以文字描述的问题。提问时可以让 AI 先描述它看到了什么，再判断原因；如果它第一步就看错了，你能及时发现。

```text
这是我电脑上的报错截图。
请先描述你看到了什么，再判断可能原因。
最后给我按顺序排查的步骤。
如果有看不清的地方，请直接说。
```

### 隐私边界

**隐私边界要按“泄露后会造成什么后果”来判断**，不能只看文件表面是否普通。

可以按风险从高到低理解：

- 最高风险：密码、验证码、`API Key`、私钥。这类内容能直接控制账号或额度。
- 高风险：证件、手机号、住址、成绩单、病历。这类内容能识别真实身份。
- 需要谨慎：客户资料、未公开合同、公司内部文档。这类内容涉及你对别人或组织的保密责任。

上传会让信息流转链条变长。原本只在你电脑或手机里的内容，会进入平台的服务器和处理系统；平台可能按自己的政策保留日志。之后只要出现账号被盗、分享链接误公开、团队权限设置错误、截图外传等情况，敏感信息就可能扩散。

如果确实要处理敏感材料，先脱敏：删除姓名、手机号、地址、账号、密钥，把真实数据替换成占位符。

**脱敏的目的，是保留任务所需结构，去掉真实身份和权限。**

例如你想让 AI 分析一份合同条款，很多时候可以把公司名改成“甲方公司 A”，把姓名改成“员工 B”，把金额改成“大约 X 元”。AI 仍然能帮你分析条款结构，但即使对话外泄，真实伤害也会小很多。

一个简单判断方法：

```text
如果这段内容被陌生人看到，我会不会有麻烦？
如果答案是会，先脱敏再上传。
```

### 小练习

找一篇公开文章或自己的学习笔记，上传后让 AI 完成三件事：

```text
请总结这份材料。
请列出我应该记住的 5 个点。
请出 3 道题检查我是否理解。
```
